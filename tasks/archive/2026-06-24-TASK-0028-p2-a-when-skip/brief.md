# Problem

Движок Owl статичен: готовность шага определяется только зависимостями `requires`, не
содержимым артефактов (`ready_resolver.rb`). Нет условных веток — нельзя выразить
«выполнять шаг X только если в brief/design выполнено условие Y». Сейчас опциональные
шаги (напр. `design`) приходится пропускать вручную (`owl step skip`). Это ограничивает
«гибкий (кастомизируемый) workflow» — заявленную цель.

# Goal

Дать шагу workflow опциональный предикат `when:`, вычисляемый по содержимому ранее
созданного артефакта. Когда предикат ложен, шаг автоматически пропускается
(`skipped`), разблокируя зависимые — без ручного `owl step skip`. Шаги без `when:`
работают как раньше (полностью обратно совместимо).

# Scenarios

### Requirement: Шаг с истинным предикатом исполняется

#### Scenario: when истинно — шаг готов к работе
- WHEN зависимости шага с `when:` выполнены И предикат (по артефакту) истинен
- THEN шаг становится `ready` и диспетчеризуется как обычный шаг

### Requirement: Шаг с ложным предикатом авто-пропускается

#### Scenario: when ложно — шаг авто-skip, зависимые разблокируются
- WHEN зависимости шага с `when:` выполнены, но предикат ложен
- THEN оркестратор авто-пропускает шаг (`status: skipped`, причина `condition_unmet`),
  и зависящие от него шаги становятся готовы (skipped разблокирует, как done)

#### Scenario: предикат по regex над артефактом
- WHEN шаг объявляет `when: { artifact: "brief", matches: "<regex>" }`
- THEN истинность = тело артефакта `brief` соответствует regex; поддержан и
  `not_matches` (отрицание)

### Requirement: Обратная совместимость

#### Scenario: шаги без when не меняются
- WHEN шаг не объявляет `when:`
- THEN его готовность/диспетч определяются только `requires`, как прежде

# Edge cases

- **Точка вычисления и доступ к данным.** `ready_resolver` чист (без доступа к
  артефактам). Предикат вычисляется в слое с `root` (orchestration/steps). `owl next`
  остаётся read-only: для готового шага с ложным `when:` он возвращает действие
  `skip_conditional_step` (как `await_plan_approval`), а мутацию (skip) делает
  оркестратор. `owl task ready-steps` помечает такой шаг отдельно (не в `ready`).
- **Расширение skip.** `owl step skip` сейчас разрешён только для `optional`-шагов.
  Разрешить авто-skip шага с ложным `when:` (валидируя предикат), даже если он не
  `optional`.
- **Отсутствие артефакта.** Если артефакт для предиката ещё не создан/отсутствует —
  предикат не вычисляется до выполнения зависимостей; если зависимость гарантирует
  артефакт, к моменту проверки он есть. Если всё же нет — безопасный дефолт (трактовать
  как «условие не выполнено» → skip, либо ошибка): зафиксировать в design.
- **Минимальный язык предикатов.** v1: `artifact` + `matches`/`not_matches` (regex по
  телу). Без сложной логики (and/or) — расширяемо позже. Невалидный regex/форма →
  понятная ошибка валидации workflow.
- **Схема.** `schemas/workflow.json` расширить полем `when` на шаге; `workflow validate`
  проверяет форму.
- **Покрытие/версия.** 100% на затронутых `**/api.rb`; minor bump + CHANGELOG.
- **Скоуп.** Только условный авто-skip одного шага. Циклы, ветвление на несколько
  путей, sub-workflow, параметры шага — вне scope (отдельная крупная работа).

# Acceptance criteria

- [ ] Шаг workflow поддерживает опциональный `when: { artifact, matches|not_matches }`;
  `schemas/workflow.json` расширена; `owl workflow validate` проверяет форму.
- [ ] Предикат вычисляется по телу указанного артефакта; истинно → шаг `ready`; ложно →
  `owl next` возвращает `skip_conditional_step`, оркестратор делает skip с причиной
  `condition_unmet`.
- [ ] `owl step skip` допускает skip шага с ложным `when:` (не только `optional`).
- [ ] `skipped` по условию разблокирует зависимые (как существующий skip).
- [ ] Шаги без `when:` — без изменений (back-compat).
- [ ] Оркестратор-skill обрабатывает `skip_conditional_step` (документировано).
- [ ] Тесты: when-true ready; when-false авто-skip + разблокировка; not_matches;
  отсутствие артефакта; невалидная форма; back-compat без when.
- [ ] `bundle exec rspec` зелёный; 100% покрытие затронутых `**/api.rb`; RuboCop net-zero.
- [ ] `Owl::VERSION` поднят + CHANGELOG.

# Out of scope
